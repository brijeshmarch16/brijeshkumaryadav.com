These are the global AI skills I use to make frontend development faster, cleaner, and more consistent.

## 1. shadcn

I use this skill when working with shadcn/ui components, project setup, component composition, and theme-friendly UI changes.

It helps me stay close to the existing design system instead of writing random custom styles.

## 2. next-best-practices

I use this skill for Next.js App Router patterns, metadata, file conventions, server components, routing, images, fonts, and production-friendly structure.

This is useful when I want the app to follow modern Next.js standards.

## 3. vercel-react-best-practices

I use this skill to think about React and Next.js performance.

It helps with bundle size, server rendering, avoiding unnecessary client code, and writing components that stay fast as the app grows.

## 4. emil-design-eng

I use this skill for UI polish and design engineering.

It helps me focus on details like spacing, visual hierarchy, motion, interaction quality, and making interfaces feel intentional.

## 5. building-components

I use this skill when creating reusable UI components.

It helps with accessibility, composable APIs, TypeScript props, design tokens, and component patterns that are easier to maintain.

These skills are useful because they give me a better workflow: one skill for framework decisions, one for UI components, one for performance, one for design polish, and one for reusable component architecture.
